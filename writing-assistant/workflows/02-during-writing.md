# 写作中检查

> **触发时机**：写作过程中 / 草稿完成后
> **目标**：确保结构完整、表达清晰

---

## 一、结构检查 ⭐⭐⭐⭐⭐

### "为什么"章节检查

```
□ 有"为什么"章节？（一票否决）
□ 认知阶段 ≥3 个理由？（一票否决）
□ 理由覆盖不同用户群体？
```

### 成功案例对照

| 维度 | ✅ 成功案例 | ❌ 失败案例 |
|------|------------|------------|
| "为什么"章节 | 有专门章节讲"为什么要用" | 直接跳到"怎么用" |
| 理由数量 | ≥3个清晰理由 | 只有1-2个模糊理由 |
| 理由质量 | 每个理由有支撑（数据/案例） | 空泛的"很有用" |

### 销售漏斗检查

```
□ 认知阶段 → 回答"为什么要用"？
□ 兴趣阶段 → 回答"适合我吗"？（≥3场景）
□ 决策阶段 → 回答"难学吗"？
□ 行动阶段 → 回答"接下来学什么"？
```

---

## 二、场景覆盖检查 ⭐⭐⭐⭐⭐

### 检查项

```
□ 有 3-5 个场景？（一票否决）
□ 覆盖 ≥3 类人群？（一票否决）
□ 场景描述具体可感知？
```

### 成功案例对照

| 维度 | ✅ 成功案例 | ❌ 失败案例 |
|------|------------|------------|
| 场景数量 | 3-5个具体场景 | 只有1个场景或无场景 |
| 人群覆盖 | 3类用户都有对应场景 | 只有单一用户视角 |
| 场景质量 | 场景可感知、有代入感 | 场景抽象、无法想象 |

### 场景示例模板

```
场景1（小白用户）：
"你是不是遇到过...[具体痛点]"

场景2（进阶用户）：
"当你想要...[更高阶目标]"

场景3（资深用户）：
"如果你正在...[专业场景]"
```

---

## 三、承诺检查 ⭐⭐⭐⭐

### 检查项

```
□ 开头有明确承诺？
□ 承诺具体可衡量？
□ 承诺可信（有支撑）？
□ 承诺有吸引力？
```

### 承诺设计原则

```
承诺 = 实际价值 × 0.8

✅ 正确：稍微低承诺，超额交付
❌ 错误：过度承诺，无法兑现
```

### 成功案例对照

| 维度 | ✅ 成功案例 | ❌ 失败案例 |
|------|------------|------------|
| 具体性 | "3分钟学会XX" | "学会XX技巧" |
| 可衡量 | 有明确数字 | 模糊的"更快更好" |
| 可信度 | 有方法论支撑 | 空泛的"非常有效" |

---

## 四、章节密度检查 ⭐⭐⭐

### 检查项

```
□ 总章节数 ≤15？
□ 每章 40-60 行？
□ 章与章之间逻辑清晰？
```

### 成功案例对照

| 维度 | ✅ 成功案例 | ❌ 失败案例 |
|------|------------|------------|
| 章节数 | ≤15章，结构清晰 | >20章，过于碎片化 |
| 章节密度 | 每章40-60行，重点突出 | 过短（<20行）或过长（>100行） |
| 逻辑连贯 | 章节间有自然过渡 | 章节跳跃，无连接 |

---

## 五、表达优化 ⭐⭐⭐⭐

### 段落块化检查

```
□ 每段 ≤5-7 行？（参考 about-me.md）
□ 每段 1 个核心观点？
□ 段落结构：观点+解释+例子？
```

### 成功案例对照

| 维度 | ✅ 成功案例 | ❌ 失败案例 |
|------|------------|------------|
| 段落长度 | 每块5-7行，易于阅读 | 大段文字，看不到重点 |
| 结构清晰 | 观点先行，例子支撑 | 绕弯子，看不懂重点 |
| 节奏感 | 长短交替，有呼吸感 | 全部长段或全部短句 |

### 开头3秒检查

```
□ 开头有痛点/对比/反常识？（必填）
□ 前3秒能抓住注意力？
□ 与承诺呼应？
```

### 开头方式

```
✅ 痛点前置："你是不是遇到过这个问题..."
✅ 对比冲击："以前需要X小时，现在只需要X分钟"
✅ 反常识："大多数人以为...其实..."
❌ 背景铺垫："随着时代的发展..."
```

---

## 六、视觉元素检查 ⭐⭐⭐⭐

### 检查项

```
□ 抽象概念有图/类比？
□ 操作步骤有截图？
□ 流程有 ASCII 可视化？
□ 图片与内容强相关？
```

### 成功案例对照

| 维度 | ✅ 成功案例 | ❌ 失败案例 |
|------|------------|------------|
| 抽象概念 | 有图/类比解释 | 全是文字，小白看不懂 |
| 操作步骤 | 每步都有截图 | 只有文字描述 |
| 流程展示 | ASCII 图/流程图 | 大段文字描述流程 |

---

## 七、术语降维检查 ⭐⭐⭐⭐

### 检查项

```
□ 首段无术语？（一票否决）
□ 必须用的术语有即时解释？
□ 降维表达符合 about-me.md 表格？
```

### 成功案例对照

| 维度 | ✅ 成功案例 | ❌ 失败案例 |
|------|------------|------------|
| 首段 | 无术语，小白能看懂 | 一堆术语，劝退用户 |
| 术语处理 | 术语+括号降维解释 | 直接使用，不解释 |
| 降维质量 | 准确、易懂、有类比 | 解释后仍然复杂 |

