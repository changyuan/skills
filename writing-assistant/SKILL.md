# 写作助手

## ⚠️ 每次使用前必读

1. 读 `about-me.md` — 了解定位、禁区、术语降维
2. 读 `user-personas.md` — 了解3类用户、4层需求

---

## 文件索引

| 文件 | 内容 |
|------|------|
| `about-me.md` | 定位、双品牌、5个主题池、禁区、术语降维 |
| `user-personas.md` | 3类目标用户、4层需求、触达策略 |
| `content/methods/01-05` | 选题→大纲→正文→标题→排版 |
| `workflows/01-04` | 写作前→写作中→发布前→发布后 |

---

## 场景路由

| 场景 | 操作 |
|------|------|
| 写新文章 | 读 `user-personas.md` 确定≥3类人群 → 执行 `workflows/01-pre-writing.md` → `02-during-writing.md` |
| 检查草稿 | 用 `workflows/02-during-writing.md` 检查，≥80分继续 → `03-pre-publishing.md` |
| 诊断数据差 | 执行 `workflows/04-post-publishing.md` → 检查对应 methods/ |
| 取标题 | 读 `methods/04-标题.md` + `workflows/03-pre-publishing.md` |
| 排版 | 读 `methods/05-排版.md` |
| 了解定位/禁区 | 读 `about-me.md` |

## 检查清单

> ≥80%发布，60-79%优化，<60%重做

### 写作前 `workflows/01-pre-writing.md`
- [ ] 目标用户 ≥3 类（参考 `user-personas.md`）
- [ ] 三维匹配定义清晰（人×情境×信息）
- [ ] 先验调研完成（小红书/知乎/微信搜一搜）
- [ ] 市场洼地判断（搜索<10条）
- [ ] 信息增量≥70%
- [ ] 未触犯 `about-me.md` 禁区

### 写作中 `workflows/02-during-writing.md`
- [ ] 有"为什么"章节 + ≥3个理由
- [ ] 3-5个场景，≥3类人群
- [ ] 承诺：具体+可信+有吸引力
- [ ] ≤15章，40-60行/章
- [ ] 抽象概念有图，操作有截图
- [ ] ≤5-7行/段
- [ ] 首段无术语（参考 `about-me.md` 术语降维表）
- [ ] 开头3秒：痛点/对比/反常识

### 发布前 `workflows/03-pre-publishing.md`
- [ ] 标题有数字对比，倍数≥10倍
- [ ] 标题有戏剧性词汇
- [ ] 标题：妈妈能理解
- [ ] 准备 ≥3 个标题备选
- [ ] 终审打分 ≥80分
- [ ] 只用 H1/H2/H3
- [ ] 有序列表无空格（`1.项目`）
- [ ] 用 `---` 换行

### 发布后 `workflows/04-post-publishing.md`
- [ ] 发布后 24-72 小时数据采集
- [ ] 问题定位清晰
- [ ] 改进建议具体可执行
- [ ] 复盘文档完成

---

## 一票否决

- 无"为什么"章节
- <3个理由
- <3类人群
- 违反 `about-me.md` 禁区
