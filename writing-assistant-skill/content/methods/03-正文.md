# 03-正文

> **核心原则**
>
> 1. **视觉优先**：图像处理50ms，文字200ms
> 2. **块化阅读**：段落≤5-7行
> 3. **术语降维**：首段无术语或有解释
> 4. **开头3秒**：痛点/对比/反常识

---

## 视觉优先

```
处理速度：
图像 50ms >> 文字 200ms

结论：抽象概念必须有图
```

**执行标准**：
```
□ 抽象概念有图？
□ 操作步骤有截图？
□ 流程有流程图？
```

---

## 块化阅读

```
1块 = 1个核心观点
1块 ≤ 5-7行

太长 → 拆分
太短 → 合并
```

**段落结构**：
```
观点（1行）
↓
解释（2-3行）
↓
例子/证据（2-3行）
```

---

## 术语降维

**原则**：首段无术语，或有即时解释

**降维模板**：
```
[术语]，就是[小白能懂的说法]

示例：
- API → "程序之间的对话接口"
- Token → "AI的阅读成本"
```

**检查**：
```
□ 首段无术语？
□ 必须用术语时有解释？
□ 用类比让小白理解？
```

---

## 开头3秒

**三种开头方式**：

### 方式1：痛点前置
```
"你是不是遇到过这个问题：[具体痛点]"
```

### 方式2：对比冲击
```
"以前需要[旧方法]，现在只需要[新方法]"
```

### 方式3：反常识
```
"大多数人以为[错误认知]，其实[正确认知]"
```

---

## 检查清单

```
□ 抽象概念有图？
□ 操作有截图？
□ ≤5-7行/段？
□ 首段无术语？
□ 开头3秒：痛点/对比/反常识？
```
